# Dataset EDA And Tuning Report

This report documents the dataset fine-tuning step after locking `merged_dataset_v3` as the baseline.

## Baseline EDA: `merged_dataset_v3`

EDA command:

```powershell
.\.venv311\Scripts\python.exe scripts\dataset_eda.py `
  --data merged_dataset_v3\data.yaml `
  --out runs\dataset_eda\merged_dataset_v3 `
  --hash-duplicates
```

Output folder:

```text
runs\dataset_eda\merged_dataset_v3
```

Conclusion: the baseline dataset is usable, but it contains invalid rows, many exact duplicates, tiny boxes, and uneven split distribution. The `glass` test set is especially weak, while `organic` is overrepresented.

## Tuned Dataset: `tuned_dataset_v1`

Build command:

```powershell
.\.venv311\Scripts\python.exe scripts\build_tuned_dataset.py `
  --data merged_dataset_v3\data.yaml `
  --out tuned_dataset_v1 `
  --overwrite
```

Output dataset config:

```text
tuned_dataset_v1\data.yaml
```

The tuned dataset is generated non-destructively. It does not modify `merged_dataset_v3`.

Tuning actions:

- Removed exact duplicate images.
- Removed unreadable or label-empty-after-cleaning images.
- Removed invalid YOLO rows.
- Removed tiny boxes smaller than `10px` or area `< 0.0002`.
- Rebuilt train/valid/test splits by dominant class.
- Applied per-class box caps to reduce overrepresentation.

The tuned version improves test coverage for weak classes. For example, `glass` increases from `200` test boxes to `863` test boxes, while invalid labels and exact duplicates are removed.

## Next Accuracy Test

The tuned dataset must be retrained before claiming any accuracy improvement.

Recommended YOLO retraining command:

```powershell
yolo detect train `
  model=runs\dl\trash_yolov8n_v3\weights\best.pt `
  data=tuned_dataset_v1\data.yaml `
  epochs=50 imgsz=800 batch=16 `
  project=runs\dl name=trash_yolov8n_tuned_v1 exist_ok=True
```

Evaluation command:

```powershell
.\.venv311\Scripts\python.exe scripts\evaluate.py `
  --weights runs\dl\trash_yolov8n_tuned_v1\weights\best.pt `
  --data tuned_dataset_v1\data.yaml `
  --out runs\dl\trash_yolov8n_tuned_v1\quality_check `
  --split both
```

Success target:

- Try to approach `0.80` mAP@0.5 or accuracy-style class performance.
- Do not claim `80%` until this retraining/evaluation produces it.

## Report Statement

Dataset fine-tuning is now evidence-based instead of random. The original dataset was analyzed with EDA, then a tuned dataset copy was generated by removing invalid labels, tiny boxes, duplicate images, and rebuilding the splits with per-class caps. The next step is retraining YOLOv8n on `tuned_dataset_v1` and comparing metrics against the locked baseline `merged_dataset_v3`.
